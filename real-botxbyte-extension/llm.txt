# BotXByte Browser Automation Extension - LLM Guide

You are interacting with a Chrome extension that executes browser automation workflows via HTTP API.
**Supports parallel execution** - multiple workflows can run simultaneously in different tabs.

## API Endpoint
POST http://localhost:8766/workflow
Content-Type: application/json

## Request Format
```json
{
  "url": "https://example.com",
  "variables": { "key": "value" },
  "options": {
    "new_tab": true,
    "close_tab": false,
    "close_tab_on_error": false
  },
  "actions": [...]
}
```

## Available Actions

### 1. wait_for - Wait for element to appear
```json
{ "type": "wait_for", "selectors": ["#element", ".fallback"], "timeout": 10000 }
```

### 2. click - Click an element
```json
{ "type": "click", "selectors": ["button.submit"] }
```

### 3. mouse_click - Click with real mouse coordinates (better for detection)
```json
{ "type": "mouse_click", "selectors": ["button.submit"] }
```

### 4. fill - Type text into input field
```json
{ "type": "fill", "selectors": ["input[name='email']"], "value": "text to type", "human_typing": true }
```
- Set `human_typing: true` for realistic typing with random delays (50-150ms per character)

### 5. keyboard - Press keyboard keys
```json
{ "type": "keyboard", "key": "Enter" }
```
- Supports: Enter, Tab, Escape, ArrowDown, ArrowUp, Backspace, etc.

### 6. delay - Wait for specified time
```json
{ "type": "delay", "timeout": 2000 }
```

### 7. navigate - Go to URL
```json
{ "type": "navigate", "url": "https://example.com/page" }
```

### 8. extract - Extract text from element into variable
```json
{ "type": "extract", "selectors": [".price"], "set_variable": "price" }
```

### 9. extract_all - Extract text from multiple elements
```json
{ "type": "extract_all", "selectors": [".product-title"], "set_variable": "titles" }
```

### 10. evaluate - Run JavaScript on page (bypasses CSP)
```json
{ "type": "evaluate", "script": "document.title", "set_variable": "title" }
```
- Use for: discovering selectors, extracting complex data, running custom logic
- Results are stored as strings (use JSON.stringify for objects/arrays)

### 11. select - Select dropdown option
```json
{ "type": "select", "selectors": ["select#country"], "value": "US" }
```

### 12. scroll - Scroll page or element
```json
{ "type": "scroll", "direction": "down", "amount": 500 }
```
- direction: "up", "down", "left", "right"

### 13. hover - Hover over element
```json
{ "type": "hover", "selectors": [".menu-item"] }
```

### 14. focus - Focus on element
```json
{ "type": "focus", "selectors": ["input#search"] }
```

### 15. clear - Clear input field
```json
{ "type": "clear", "selectors": ["input#search"] }
```

### 16. double_click - Double click element
```json
{ "type": "double_click", "selectors": [".item"] }
```

### 17. screenshot - Take screenshot (base64)
```json
{ "type": "screenshot", "set_variable": "screenshot_data" }
```

### 18. if_exists - Conditional action if element exists
```json
{ "type": "if_exists", "selectors": [".popup"], "timeout": 2000, "actions": [
  { "type": "click", "selectors": [".close-popup"] }
]}
```

### 19. if_visible - Conditional action if element is visible
```json
{ "type": "if_visible", "selectors": [".cookie-banner"], "timeout": 2000, "actions": [
  { "type": "click", "selectors": [".accept-cookies"] }
]}
```

### 20. set_variable - Set a variable manually
```json
{ "type": "set_variable", "name": "status", "value": "completed" }
```

### 21. iframe - Execute actions inside iframe
```json
{ "type": "iframe", "selectors": ["iframe#payment"], "actions": [...] }
```

## Selector Types

### CSS Selectors (default)
```json
"selectors": ["#id", ".class", "input[name='email']", "[data-testid='button']"]
```

### XPath Selectors (prefix with xpath:)
```json
"selectors": ["xpath://div[@class='price']/span"]
```

### Text Content Match (prefix with text:)
```json
"selectors": ["text:Submit Order"]
```
- Case-insensitive, matches elements containing the text

## Variables
- Define in `variables` object: `"variables": { "email": "test@example.com" }`
- Use in values with `${name}`: `"value": "${email}"`
- Extract results with `set_variable` property
- Variables persist across actions in workflow

## Discovering Selectors
Before writing a workflow, discover page structure:
```json
{
  "url": "https://target-site.com",
  "actions": [
    { "type": "delay", "timeout": 3000 },
    { "type": "evaluate", "script": "JSON.stringify([...document.querySelectorAll('input, button, a, [role]')].slice(0,30).map(e => ({tag: e.tagName, id: e.id, class: e.className, name: e.name, placeholder: e.placeholder, text: e.innerText?.slice(0,50), href: e.href?.slice(0,50), role: e.getAttribute('role'), dataAttrs: Object.keys(e.dataset)})))", "set_variable": "elements" }
  ]
}
```

## Example Workflows

### Search on a website
```json
{
  "url": "https://www.nike.in",
  "actions": [
    { "type": "wait_for", "selectors": ["input[data-at='search-input']"], "timeout": 10000 },
    { "type": "click", "selectors": ["input[data-at='search-input']"] },
    { "type": "fill", "selectors": ["input[data-at='search-input']"], "value": "air jordan", "human_typing": true },
    { "type": "keyboard", "key": "Enter" }
  ]
}
```

### Extract data from page
```json
{
  "url": "https://ahrefs.com/website-authority-checker/",
  "variables": { "domain": "example.com" },
  "actions": [
    { "type": "wait_for", "selectors": ["input[placeholder='Enter domain']"], "timeout": 10000 },
    { "type": "fill", "selectors": ["input[placeholder='Enter domain']"], "value": "${domain}", "human_typing": true },
    { "type": "mouse_click", "selectors": ["button[type='submit']"] },
    { "type": "wait_for", "selectors": [".ReactModalPortal [role='dialog']"], "timeout": 15000 },
    { "type": "delay", "timeout": 2000 },
    { "type": "extract", "selectors": ["xpath://div[contains(text(),'Domain Rating')]/ancestor::div[contains(@class,'css-1fu38of')]//span[contains(@class,'css-vyilnr')]"], "set_variable": "domain_rating" }
  ]
}
```

### Handle popups/modals
```json
{
  "url": "https://example.com",
  "actions": [
    { "type": "delay", "timeout": 2000 },
    { "type": "if_exists", "selectors": [".cookie-consent"], "timeout": 3000, "actions": [
      { "type": "click", "selectors": ["button.accept-cookies"] }
    ]},
    { "type": "if_visible", "selectors": [".newsletter-popup"], "timeout": 2000, "actions": [
      { "type": "click", "selectors": [".close-popup", "button[aria-label='Close']"] }
    ]}
  ]
}
```

### Login flow
```json
{
  "url": "https://example.com/login",
  "variables": { "username": "user@example.com", "password": "secret123" },
  "actions": [
    { "type": "wait_for", "selectors": ["input[name='email']", "#email"], "timeout": 10000 },
    { "type": "fill", "selectors": ["input[name='email']", "#email"], "value": "${username}", "human_typing": true },
    { "type": "fill", "selectors": ["input[name='password']", "#password"], "value": "${password}", "human_typing": true },
    { "type": "mouse_click", "selectors": ["button[type='submit']", "text:Sign In"] },
    { "type": "wait_for", "selectors": [".dashboard", "[data-page='home']"], "timeout": 15000 }
  ]
}
```

## Response Format
```json
{
  "id": "uuid",
  "success": true,
  "message": "Workflow completed",
  "variables": { "extracted_var": "value" },
  "logs": [
    { "time": "ISO timestamp", "type": "action", "message": "click -> SUCCESS" }
  ]
}
```

## Best Practices

1. **Always use wait_for first** - Ensure elements exist before interacting
2. **Use multiple selectors** - Provide fallbacks: `["#id", ".class", "[data-attr]"]`
3. **Use human_typing for inputs** - Reduces bot detection: `"human_typing": true`
4. **Use mouse_click for buttons** - More realistic than programmatic click
5. **Add delays after navigation** - Pages may load content dynamically
6. **Use if_exists/if_visible for popups** - Handle optional elements gracefully
7. **Discover selectors first** - Use evaluate to find correct selectors before writing workflow
8. **Check data-* attributes** - Sites use various naming: `data-testid`, `data-at`, `data-qa`, etc.

## Limitations
- Extension must be loaded in Chrome
- Server (server.py) must be running
- Some sites may detect automation via `isTrusted: false` on events
- Page navigation breaks content script connection (expected behavior)

## Parallel Execution

Multiple workflows can run simultaneously. Each workflow:
- Gets its own tab (when `new_tab: true`)
- Has isolated variables
- Runs independently

### Run workflows in parallel (bash):
```bash
# Start 3 workflows simultaneously
curl -X POST http://localhost:8766/workflow -d @workflow1.json &
curl -X POST http://localhost:8766/workflow -d @workflow2.json &
curl -X POST http://localhost:8766/workflow -d @workflow3.json &
wait
```

### Check running workflows:
```bash
curl http://localhost:8766/status
# Returns: {"running": 3, "workflows": ["id1", "id2", "id3"]}
```

### Cancel a workflow:
```bash
curl -X DELETE http://localhost:8766/workflow/{workflow_id}
```
